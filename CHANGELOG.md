# 更新日志

本项目所有值得注意的变更都记录在这个文件里。

格式遵循 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，版本号遵循[语义化版本](https://semver.org/lang/zh-CN/)（`主版本.次版本.修订号`：不兼容改动 / 新功能 / 修 bug）。

维护约定：
- 平时改代码，把变更随手记到 `[Unreleased]` 段
- 发版时把 `[Unreleased]` 改成版本号 + 日期，开新的空 `[Unreleased]`
- GitHub Release 的发布说明直接复制对应版本的段落

## [1.11.0] - 2026-06-13

### Added
- 代码/文本编辑器改成「和 md 一致」：去掉「保存 / 完成」按钮，停笔自动落盘；顶部加**撤销 / 重做**两个图标（按撤销栈可用性灰显），「保存」状态实时显示**「xx 之前已保存」**（1 分钟内显秒、1 小时内显「分:秒」、再久显最后保存钟点）
- **双击终端顶栏空白处 = 终端铺满**：agent 交互窗口一键放到最大，挤掉文件区和预览
- 侧栏「收藏」「AGENT 项目」列表也会高亮**当前所在目录**（原先只有「快速入口」有指示），重渲后保持选中态

### Changed
- 折叠 / 展开侧栏时，腾出 / 收回的宽度在「文件区+预览」和终端之间**按比例分配**，不再全甩给左侧文件区（左右分栏）
- 默认设置调整（仅对全新 profile 生效）：默认皮肤「档案」、文件图标最小尺寸、终端首次**左右分栏**且文件区:终端 = **1:2**、文件区:预览 = **1:2**（拖过 resizer 后用记下的 px）
- 移除「文件区最小尺寸」自动挤压策略：导航不再强行改你的分栏比例。仅保留一个特例——终端铺满 / 全铺时主动导航会退出该状态，让文件区回来

### Fixed
- 预览全屏时左上角操作按钮（如 md「源码」）点不动：顶栏 `-webkit-app-region: drag` 是 OS 级遮罩、不认 z-index，会吞掉压在上面的预览按钮点击。全屏期间关掉顶栏拖拽区
- 预览全屏时隐藏 macOS 左上角红黄绿系统按钮（和右侧自家关闭图标太像，易误点），退出全屏恢复
- **图片全屏在窗口中居中**，整图缩放铺满可视区（约束 max-height 避免高图顶部被裁）
- **压缩包内中文文件名乱码**：不再依赖系统 unzip（会按自己的逻辑转码丢失编码），改为直接读 zip 中央目录，按 ZIP「通用位标记 bit 11」判断 UTF-8、否则按 GBK 解码

## [1.10.0] - 2026-06-13

### Added
- 终端里点 md/html 路径：定位打开后**直接全屏预览**，最贴合「点一下就想看清这文件长啥样」的意图；代码等其它类型仍退回常规分栏
- 文件区**双击文本/代码 = 全屏预览**（单击仍是分栏轻预览）；图片/视频等双击仍交系统默认 App 打开

### Changed
- **预览即编辑**：代码/纯文本一进预览区就是可编辑态（和 md 一致），不用再点编辑按钮；html（看的是渲染形态）、csv/tsv（表格视图）保持只读展示

### Fixed
- 文件跟随追到原子写临时文件（`foo.swift.tmp.<pid>.<hex>` 这类 `.tmp` 在中段的命名）后报 ENOENT、右上角挂着「跟随中 …tmp…」的怪状态：临时文件过滤从「`.tmp` 结尾」放宽到「`.tmp` 在结尾或后面还跟东西」，从源头拦掉

## [1.9.1] - 2026-06-13

### Added
- 文件跟随「过程旁白」（实时舞台阶段二/三）：预览主视图仍是 agent 产出的成品，底部加一行实时旁白说 agent 此刻在干嘛。从绑定终端 tab 的输出尾巴提炼动作（认 Claude Code 的 ⏺ Update/Bash/Read/Grep 工具行和 Web Search），翻成「写 X / 跑 Y / 搜 Z / 联网搜 W」；忙时「agent 正在 …」配脉冲点，闲下来「停在 X」配静止点。只认绑定 tab 的输出，不串别的终端
- 截图/文件可拖进文件区落盘：拖到空白处存进当前目录，拖到某文件夹图标上存进那个文件夹。macOS 截图浮窗缩略图（没磁盘路径）和 Finder 文件（有路径）都认；同名不覆盖（仿访达存成 foo 2.png）；落地有高亮提示

## [1.9.0] - 2026-06-13

### Added
- 预览全屏放大：顶栏一颗按钮把预览铺满整个窗口（盖住文件区/终端/侧边栏），Esc 或再点退出。md 全屏下依然所见即所得、可继续编辑；html/长文档放大后看着舒服
- HTML 预览支持完整交互：点击、键盘、localStorage、fetch、表单、SPA 全部能跑。靠「隔离预览源」做到安全——新增预览专用端口（主端口+1）只出文件、不挂 /api，预览页与 App 跨源，碰不到 App 本体也摘不掉 sandbox；可读范围收紧到主目录、挡掉点目录防敏感文件外泄

### Changed
- 文件跟随「钉死归属」：跟随死死锚定到你选的那个终端 tab，只盯这个 agent。归属从「路径前缀猜」升级成「路径 + agent 此刻在不在干活」双重判断，多 agent 多终端时别的 tab 的写入不再串进来。开跟随必须绑一个活动终端（没有就提示），不再退化成「全文件系统跟随」。UI 明确显示在跟哪个项目/agent：徽标带项目名、被盯的 tab 描边高亮 + 眼睛标记
- 文件跟随产物不再霸屏：agent 编译/打包出的二进制、压缩包、安装包（.app 内部、无扩展名可执行、.dylib/.zip/.dmg 等）不再被当文本跟随、显示「无法预览」，改成一张干净的产物卡片（图标 + 「agent 刚生成 · 大小」+ 在访达显示/打开）
- 底部少两条多余分割线：文件区和预览的底部信息栏不再各带横线（本就有背景色差自分隔），底部只剩终端那一条可拖的边界，不再有「拖哪条」的困惑

### Fixed
- 删掉 HTML 预览里那颗空着的没用小按钮（原是「适配宽度/实际大小」切换钮，因全局缺 .hidden 规则一直空露在外）；保留自动缩放适配预览框宽度的行为

## [1.8.2] - 2026-06-12

### Fixed
- 文件跟随四处失控一并修掉：
  - agent 只是「读」文件也触发跟随和卡片闪烁——macOS FSEvents 连 atime/元数据更新都报。主进程转发事件前先 stat，mtime/ctime 都不新鲜（内容根本没动）的事件直接丢弃，读文件从此安静
  - 隐藏文件（.env、.git 配置等）和 sqlite 临时 sidecar（`-journal`/`-shm`/`-wal`/`.tmp`/`.lock`）也被跟随——用户什么都没的看。统一进噪声过滤，跟随/闪卡/变更收件箱全部不再理会
  - 跟随范围曾是「所有监听目录」：浏览到 ~/.cc-switch 时别的 App 写 db-journal 也被跟过去，噪声还在节流窗口里不断顶掉真正该看的 html/md。改为开启时绑定当前终端 tab，只跟它项目目录（cwd 随 agent 走）下的写入；切 tab 不跟别的 agent，绑定的 tab 关掉跟随自动停
  - 节流窗口内加看头优先级：html/md（写给人看的）> 代码 > 其它，低优先级写入不再顶掉排队中的高优先级目标
  - 关闭跟随时所有动作就地停：in-flight 的 html 双缓冲换页补上 follow 状态守卫，停掉后不再抢屏

## [1.8.0] - 2026-06-12

## [1.7.3] - 2026-06-12

### Changed
- 编辑图片时面板撑满整窗：原来编辑器挤在小预览栏里，画布太小没法精细标注。进入编辑即全屏（画布居中放大到视口最大），保存/关闭/切文件自动还原。纯 CSS `:has()` 实现——感知编辑器 DOM 是否在面板里，不靠 JS 维护全屏状态，4 个退出路径都不可能漏还原

## [1.5.2] - 2026-06-11

### Changed
- 顶栏防挤压整修：右侧操作区不再参与收缩（挤压压力交给可滚动的面包屑），图标在任何宽度下不变形
- 「变更」「最近」改为纯图标按钮（悬停有提示）；「变更」数字徽标改成小圆点，有变更才亮，不再频繁跳数
